# পাথের উপর দিয়ে অতিক্রম করে

মৌলিক বৈশিষ্ট্য tsoa কোন শক্তি প্রদান করে না, এর ফলে এক শক্তিশালী (কিন্তু ব্যয়বহুল পদ্ধতি) প্রদান করা হয় tsoa রুট নির্মাণের সময় ব্যবহারযোগ্য স্বনির্ধারিত ফর্মা প্রয়োগ করা হবে।

::: danger
একটি কাস্টম টেমপ্লেট ব্যবহার করে এমন একটি নতুন সংস্করণে মাইগ্রেট করা আরও কঠিন হবে tsoa যেহেতু তোমার টেমপ্লেটের সাথে সম্পর্কিত tsoa অভ্যন্তরীণ. তো, নতুন আর ভালো বৈশিষ্ট্য পেতে tsoaঅনুগ্রহ করে আপনার নির্বাচিত প্রিন্টারের জন্য বিভিন্ন টেমপ্লেট দ্বারা প্রয়োগ করুন `"middleware"` (যেমন "কা", "মো", অথবা "হাপি") `"middlewareTemplate"`. .
:::

ঠিক আছে, কিন্তু আপনি কেন রুট টেমপ্লেট উপেক্ষা করতে চান? নতুন (_N)

- আপনি কি সার্ভার কাঠামো ব্যবহার করছেন যা এখনও সমর্থন করে না? যদি তাই হয়, তাহলে [please open an issue first](https://github.com/tsoa-next/tsoa-next/issues). . নতুন আদর্শ অপশন হিসেবে আমরা আপনার স্বনির্বাচিত টেমপ্লেটকে মেনে নেব। নতুন কাঠামোকে আমরা সমর্থন করতে না পারলে, আমরা একটি স্বনির্ধারিত রুট টেমপ্লেট সুপারিশ করবো।
- আপনার কি একটা নির্দিষ্ট চাহিদা আছে? আপনার কাছে ইতিমধ্যেই একটি সমস্যা আছে এবং আছে tsoa রক্ষণাবেক্ষণকারী এই বৈশিষ্ট্য সমর্থন করার জন্য সিদ্ধান্ত নিয়েছে? তাহলে একটি স্বনির্ধারিত টেমপ্লেট আপনার প্রয়োজন প্রয়োজন সমাধান করতে পারে.

পূর্বতনের মাধ্যমে পূর্বতন টেমপ্লেট থেকে নেয়া হয়। নিজের টেমপ্লেট প্রয়োগ করে নিজের টেমপ্লেট নির্ধারণ করা যাবে এবং এটি ব্যবহার করতে পারবেন
এটা আপনার ভাষায় tsoa..jon কনফিগারেশন. মধ্যবর্তী জায়গায় রুট পথ স্থাপন করা হয়েছে, যার উপর ভিত্তি করে আপনি উল্লেখ করেছেন।

```js
{
  "entryFile": "...",
  "spec": {
    ...
  },
  "routes": {
    "routesDir": "...",
    "middleware": "express",
    "middlewareTemplate": "custom-template.ts",
    ...
  }
}
```
