# FAQ

## পারে আমি ব্যবহার করি OpenAPI ৩ বা ৩ এর পরিবর্তে OpenAPI ২ (পর্যন্ত) Swagger- তাই?

হ্যাঁ। সংকলন `spec.specVersion` চিহ্নিত স্থানে চলুন `3` অথবা `3.1` তোমার মধ্যে `tsoa.json` ফাইল. অতিরিক্ত বিকল্প অনুসন্ধান করুন [`Config`](./reference/tsoa-next/interfaces/Config.md) API রেফারেন্স.

## আমি কিভাবে ব্যবহার করব tsoa কোরা, কিংবা অন্য কাঠামো দিয়ে?

আপনার মধ্যেHiling- এর ঘর নির্ধারণ করুন tsoa কনফিগারেশন. বাক্সের বাইরে, এক্সপ্রেস, ওসি এবং কোরা সমর্থিত।
আরো তথ্যের জন্য আপনি একটি স্বনির্ধারিত টেমপ্লেটও দিতে পারেন, অনুগ্রহ করে যাচাই করুন [the guide](./templates.md)

## কিভাবে অতিরিক্ত সম্পত্তি ব্যবহার করতে হবে

ডিফল্ট হিসাবে, OpenAPI মডেল হতে দেয় [`additionalProperties`](https://swagger.io/docs/specification/data-models/dictionaries/). . আপনি যদি এ বিষয়ে নিশ্চিত করতে চান যে, আপনার মডেলে শুধুমাত্র আপনার ব্যবস্থাপনার জন্য নির্ধারিত বৈশিষ্ট্য নির্ধারণ করা হবে, তাহলে সেই তথ্য নির্ধারণ করুন `noImplicitAdditionalProperties` বিকল্প [`Config`](./reference/tsoa-next/interfaces/Config.md) হয় হয় `"silently-remove-extras"` অথবা `"throw-on-extras"`. .
গুহা:

- নিম্নলিখিত ধরনের ধরন, তারা যেভাবে কাজ করে থাকে, তার জন্য অতিরিক্ত সম্পত্তিকে সবসময় সুযোগ দেওয়া হবে:
  - এটা `any` ধরন
  - একটি তুলনার ধরন (যে এটিকে সরাসরি অতিরিক্ত বৈশিষ্ট্যাবলী অনুমোদন করে) `export interface IStringToStringDictionary { [key: string] : string }`- যদি আপনি ব্যবহার করেন tsoa এমন একটি সেবা যা ভোক্তার কাছে আছে.
  - সেট করার আগে আপনাকে আপনার ক্রেতাকে জানাতে হবে `noImplicitAdditionalProperties` চিহ্নিত স্থানে চলুন `"throw-on-extras"` যেহেতু এটি একটি ভাঙ্গা পরিবর্তন হবে (কারণ এর আগে কাজ করা মৃতদেহগুলো এখন কোন ভুল খবর পাবে)।
- যাই হোক, `"noImplicitAdditionalProperties" : "silently-remove-extras"` উত্তরাধিকার এবং নতুন APIের জন্য একটি চমৎকার পছন্দ (সি # সিরিয়ালাইজার এবং অন্যান্য জনপ্রিয় JSON সিরিয়ালারের আচরণ)।

## অনুরূপ সংস্করণের সাথে মন একত্রিত করার চেষ্টা করুন

একই নামের একাধিক মডেল উপস্থিত থাকলে, আপনি হয়ত একাধিক মিল বিশিষ্ট মডেল উৎপন্ন করতে পারেন। আপনি যদি একটি মডেল এর 'কাননকাল' সংস্করণ হিসাবে একটি শ্রেণী/বিন্যাস উল্লেখ করতে চান, তাহলে এটিকে একটি জিএলডি উপাদান হিসেবে যোগ করুন:

```ts
/**
 * @tsoaModel
 */
export interface MyModel {
  ...
}
```

## আমার ওএসা থেকে আমি কিভাবে সবচেয়ে বেশি পেতে পারি?

এখন আপনি একটি আছে OpenAPI প্রণালী (ওএসএএস), আপনি বিভিন্ন ধরনের বিস্ময়কর যন্ত্রপাতি ব্যবহার করতে পারেন যা ডকুমেন্টস, ক্লায়েন্ট এসডিকেস এবং আরও অনেক কিছু তৈরি করতে পারে। [here](http://openapi.tools). .

## যে ভাবে বড় অ্যারের জন্য সুনির্দিষ্ট সীমা অতিক্রম করে (যেমন ২০ এর বেশি)

ডিফল্ট [Express](https://github.com/expressjs/express) ব্যবহৃত হচ্ছে [qs](https://github.com/ljharb/qs) ছোট করে দেখার জন্য ডিফল্ট মান ধার্য হয়েছে, এবং ছবির জন্য নির্ধারিত রয়েছে ২০টি উপাদান।
এই কনফিগারেশনের ক্ষেত্রে নিম্নলিখিত কনফিগারেশন প্রয়োগ করা আবশ্যক:

```ts
const app = express()

app.set('query parser', function (str) {
  return qs.parse(str, { arrayLimit: Infinity })
})

app.use(bodyParser.json())
app.use(Router())
```

নোট দিন যে, আপনি এটা অন্যান্য অপারেটিং সিস্টেমের উপরে রাখতে হবে ।
