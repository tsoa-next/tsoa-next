# Преобладающий шаблон маршрута

Если вам нужна функциональность, которая tsoa не обеспечивает, то один мощный (но потенциально дорогостоящий подход) должен обеспечить tsoa с пользовательским шаблоном ручек для использования при создании файла route.ts.

::: danger
Использование пользовательского шаблона означает, что вам будет сложнее перейти на новые версии. tsoa Поскольку ваш шаблон взаимодействует с tsoa Внутренние. Итак, чтобы получить новейшие и лучшие характеристики tsoaПожалуйста, используйте один из предоставленных шаблонов, выбрав выбранный вами `"middleware"` (т.е. «коа», «экспресс» или «хапи») и путем опущения `"middlewareTemplate"`.
:::

Хорошо, но почему вы хотите переопределить шаблон маршрута? _

- Вы используете серверную структуру, которую мы еще не поддерживаем? Если это так, то [please open an issue first](https://github.com/tsoa-next/tsoa-next/issues). Вполне вероятно, что мы постараемся принять ваш пользовательский шаблон в качестве одного из новых стандартных вариантов. Если мы не можем поддерживать новую структуру, то мы порекомендуем пользовательский шаблон маршрута.
- У вас есть конкретное требование? Вы уже открыли вопрос и имеете tsoa Разработчики решили не поддерживать эту функцию? Тогда пользовательский шаблон может наилучшим образом решить ваши потребности.

Шаблоны маршрута генерируются из предварительно заданных шаблонов рульбара. Вы можете переопределить и определить свой собственный шаблон для использования
Определяя его в своей tsoaКонфигурация .json. Маршрутные пути генерируются на основе типа промежуточного программного обеспечения, который вы определили.

```js
{
  "entryFile": "...",
  "spec": {
    ...
  },
  "routes": {
    "routesDir": "...",
    "middleware": "express",
    "middlewareTemplate": "custom-template.ts",
    ...
  }
}
```
