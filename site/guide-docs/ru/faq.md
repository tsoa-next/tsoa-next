# FAQ

## может Я использую OpenAPI 3 или 3.1 вместо OpenAPI 2 (ранее) Swagger)?

Да. Настройка `spec.specVersion` то `3` или `3.1` в твоем `tsoa.json` Файл. Больше опций конфигурирования в [`Config`](./reference/tsoa-next/interfaces/Config.md) Ссылка на API.

## Как я использую tsoa с коа, хапи или другими структурами?

Установите свойство промежуточного программного обеспечения в вашем tsoa конфигурация. Из коробки поддерживаются экспресс, хапи и коа.
Вы также можете предоставить пользовательский шаблон, для получения дополнительной информации, пожалуйста, проверьте [the guide](./templates.md)

## Как обеспечить отсутствие дополнительных свойств во время выполнения

По умолчанию, OpenAPI позволяет моделям иметь [`additionalProperties`](https://swagger.io/docs/specification/data-models/dictionaries/). Если вы хотите убедиться во время выполнения, что данные имеют только свойства, определенные в ваших моделях, установите `noImplicitAdditionalProperties` вариант в [`Config`](./reference/tsoa-next/interfaces/Config.md) либо `"silently-remove-extras"` или `"throw-on-extras"`.
Пещеры:

- Следующие типы всегда допускают дополнительные свойства из-за характера их работы:
  - The `any` тип
  - Индексированный тип (который явно допускает дополнительные свойства). `export interface IStringToStringDictionary { [key: string] : string }`- Если вы используете tsoa Существующий сервис, имеющий потребителей...
  - Вам нужно будет сообщить своим потребителям перед установкой `noImplicitAdditionalProperties` то `"throw-on-extras"` так как это было бы сокрушительным изменением (в связи с тем, что запросы органов, которые ранее работали, теперь получили бы ошибку).
- Несмотря ни на что. `"noImplicitAdditionalProperties" : "silently-remove-extras"` Это отличный выбор как для устаревших, так и для новых API (поскольку это отражает поведение C#-сериализаторов и других популярных JSON-сериализаторов).

## Работа с дублирующими именами моделей

Если у вас есть несколько моделей с одним и тем же названием, вы можете получить ошибки, указывающие на то, что существует несколько совпадающих моделей. Если вы хотите обозначить класс / интерфейс как «каноническую» версию модели, добавьте элемент jsdoc, обозначающий его как таковой:

```ts
/**
 * @tsoaModel
 */
export interface MyModel {
  ...
}
```

## Как переопределить лимит для проверки больших массивов (с более чем 20 элементами)

По умолчанию [Express](https://github.com/expressjs/express) использование [qs](https://github.com/ljharb/qs) как парсер внутри, и имеет ограничение по умолчанию для проверки 20 элементов в массиве
Чтобы преодолеть это, вы должны добавить следующую конфигурацию в свой экспресс-конфигурку:

```ts
const app = express()

app.set('query parser', function (str) {
  return qs.parse(str, { arrayLimit: Infinity })
})

app.use(bodyParser.json())
app.use(Router())
```

Обратите внимание, что вы должны поместить его поверх другого промежуточного программного обеспечения.
